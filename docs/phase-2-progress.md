## Tasks (will be checked off as completed)

See the plan for the canonical task list. Progress updates land here per push so a PR reviewer can scan a single file for context.

- [x] **Task 1** — MarkdownUI + swift-snapshot-testing wired into `MatronShared/Package.swift`; added `DesignSystemSnapshotTests` test target. `MatronDesignSystem` was already declared in `project.yml` from Phase 1, so no project-yml churn.
- [x] **Task 2** — `MarkdownText` primitive shipped with `Theme.matron` (system font, monospaced inline code, copy-button code blocks, accent-coloured underlined links). 4 snapshot tests × 3 macOS variants = 12 baselines recorded. Plan deviation: `swift-snapshot-testing` only ships a SwiftUI-aware `.image` strategy on iOS/tvOS, so on macOS the `assertVariants` helper hosts the SwiftUI view in `NSHostingView` and snapshots that. iOS variants will be wired up when this suite gets a host xcodebuild scheme.
- [x] **Task 2b** — Promoted private `CodeBlockView` to public `CodeBlock` primitive in its own file; `MarkdownText`'s `.codeBlock` theme hook now delegates to it. 2 snapshot tests × 3 macOS variants = 6 new baselines. Existing MarkdownText baselines unchanged (byte-identical render).
- [x] **Task 3** — `AttachmentImage` (image-with-placeholder + caption) and `AttachmentFile` (icon + filename + ByteCountFormatter size) primitives shipped. 3 snapshot tests × 3 macOS variants = 9 new baselines. Per the plan note, `AttachmentFile`'s background uses `Color.matronCodeBg` (cross-platform alias from MarkdownText) instead of iOS-only `Color(.systemGray6)`.
- [x] **Task 4** — `TimelineItem` DTO shipped with `Kind` (text/image/file/stateChange/unknown) and `SendState` (sent/sending/failed) enums. 5 unit tests cover equality, kind disjointness, stable id, default sendState, and failure-reason equality.
- [x] **Task 5** — `TimelineService` protocol + `TimelineServiceLive` (SDK-backed) + `FakeTimelineService` shipped. 5 fake-driven tests cover snapshot streaming, sendText/Image/File call-recording, paginate/markAsRead. Plan deviations to match SDK v26 (26.04.01): `messageEventContentFromMarkdown(md:)` is a single-arg call (no html); `sendImage`/`sendFile` take `UploadParameters` + `thumbnailSource:` (not `url:thumbnailUrl:`); `paginateBackwards(numEvents:)` (not `opts:`); `markAsRead` lives on `Timeline` not `Room`; SDK exports `TimelineListener` (no `Protocol` suffix). The diff listener also handles `.popFront`/`.popBack`/`.truncate` cases the plan omitted.
- [x] **Task 5b** — Timeline diff application is implemented as `TimelineSnapshotListener.onUpdate(diff:)` in `TimelineServiceLive` (Task 5). `SnapshotApplier` is the test-only mirror that runs the same switch statement over a `FakeDiff` enum. 19 tests cover all 11 SDK `TimelineDiff` cases (including the v26-only `popFront`/`popBack`/`truncate`), the local-echo → remote-event id swap, out-of-bounds clamping, and unknown-event preservation. Also dropped the stale `MatronShared/Sources/DesignSystem/_Module.swift` placeholder.
- [x] **Task 6** — `ChatService.createChat(with:)` extends the protocol; `ChatServiceLive.createChat` builds a `CreateRoomParameters` (encrypted, direct, private preset, single-bot invite) and delegates to `Client.createRoom`. 2 fake-driven tests exercise the protocol surface; existing fakes (`FakeChatService` + `MatronMacTests/LocalFakeChatService`) updated for the new requirement. Plan call site matches v26 (26.04.01) `CreateRoomParameters` exactly.
- [x] **Task 7** — `BotCommand` model + `BotCommandCatalog.claudeBridge` (14 commands) shipped in `MatronModels`. `filter(_:byPrefix:)` is case-insensitive and tolerant of either `/` or `!` prefix. 5 tests cover empty prefix → all, prefix matches, bang prefix, no-match → empty, and uniqueness of triggers in the catalog.
- [x] **Task 8** — `ComposerViewModel` (@Observable @MainActor, no deinit) shipped in `MatronViewModels`. State: `input`, `isSending`, `sendError`, `palettePinnedOpen`. Computed `showPalette` (single-token `/`/`!`) + `filteredCommands`. `send()` trims, clears on success, preserves input + records error on failure. `attachFiles(_:)` is the Mac drag-drop entry point — sends image/file by MIME prefix. 7 tests cover palette show/hide, single-token boundary, command selection, send happy path, empty-input no-op, and send-error preservation.
- [x] **Task 9** — `ComposerView` + `SlashCommandPalette` + `AttachmentPicker` shipped under `Matron/Features/Chat/Composer/`. Plan deviation: the plan deferred `fileImporter` + `PhotosPicker` wiring to Task 11 ("Wired in Task 10 once Live timeline supports sendFile"), but `ComposerViewModel.attachFiles(_:)` and `TimelineService.sendImage`/`sendFile` already exist (Tasks 5 + 8), so the photo picker and file importer route through `attachFiles(_:)` immediately — no dead state. 2 binding tests in `MatronTests/ComposerViewBindingTests.swift` cover input → send wiring and command-list passthrough to the palette.
- [x] **Task 10** — `ChatViewModel` (@Observable @MainActor, no deinit) shipped in `MatronViewModels`. State: `roomID`, `items`, `error`. `start()` returns the observation `Task` so tests await drain deterministically (no sleeps). `stop()` cancels for `View.onDisappear`; `paginateBackward`/`markAsRead`/`refresh` forward to `TimelineService`. 7 tests cover snapshot streaming, deterministic completion (2s bound), paginate/markAsRead invocation, refresh-routes-to-paginate, no-error-on-success, and stop-idempotency. Reused the existing `FakeTimelineService` from `ComposerViewModelTests.swift` instead of duplicating it.
