## Bugbot pass — fixed 8 issues across composer / timeline / chat-vm

Cleared all 8 bugbot findings from the Phase 2 batches 3–5 review before Task 14 piles on top. One commit on `phase-2-chat-experience`. Per-bug summary:

1. **Composer palette reopens after command selection** (`MatronShared/Sources/ViewModels/ComposerViewModel.swift`) — `showPalette` now drops only leading whitespace instead of trimming both sides, so the trailing space `selectCommand(_:)` adds (`"/start "`) keeps the palette closed. New tests: `test_palette_staysClosed_afterCommandSelection`, `test_palette_isHiddenForCommandWithTrailingSpace`.
2. **Photo picker MIME wrong** (`Matron/Features/Chat/Composer/AttachmentPicker.swift` + `ComposerView.swift`) — `PhotosPicker` now initialises with `photoLibrary: .shared()`, and the selection handler picks the file extension from the item's `supportedContentTypes` (HEIC stays HEIC, PNG stays PNG) instead of falling back to `"photo.jpg"` and forcing `image/jpeg`.
3. **Send button enabled on whitespace-only** (`Matron/Features/Chat/Composer/ComposerView.swift`) — added `isSendable` (mirrors `send()`'s `.whitespacesAndNewlines` trim) and used it for both `.disabled(...)` and the accent-colour styling. New tests: `test_isSendable_matchesSendBehaviour_forWhitespaceOnly`, `test_send_doesNothing_forWhitespaceOnlyInput_andSendErrorStaysNil`.
4. **fileImporter URLs need security-scoped access** (`Matron/Features/Chat/Composer/ComposerView.swift` + `MatronShared/Sources/ViewModels/ComposerViewModel.swift`) — added `stageAndAttach(_:)` that wraps each URL with `start/stopAccessingSecurityScopedResource()` and stages bytes into a temporary file before calling `attachFiles(_:)`. Removed the silent `try?` in `attachFiles(_:)`; failures now surface via `sendError`. Added `ComposerViewModel.reportAttachmentError(_:)` so view-layer staging errors can flow through the same field while keeping `sendError` `private(set)`. New test: `test_reportAttachmentError_recordsSendError`.
5. **Display name has `@` prefix** (`Matron/Features/Chat/Rendering/TimelineItemView.swift`) — `displayName(for:)` now drops the leading sigil so `"@bot:server.com"` → `"bot"`. Promoted to `static internal` and tested directly. New file: `MatronTests/TimelineItemViewTests.swift` (4 tests covering happy path, missing sigil, no-colon malformed input, `@`-only edge case).
6. **Listener handle race / leak** (`MatronShared/Sources/Chat/TimelineServiceLive.swift`) — replaced double-assignment of `continuation.onTermination` with `TimelineLifecycleHolder`, which atomically owns both the setup `Task` and the SDK listener `TaskHandle` and cancels both on termination. Late-arriving registrations self-cancel if termination already fired. New file: `MatronShared/Tests/ChatTests/TimelineLifecycleHolderTests.swift` (4 tests).
7. **`myID` dead** (`MatronShared/Sources/Chat/TimelineServiceLive.swift`) — dropped the unused `myID` stored property and init parameter from `TimelineSnapshotListener` (the SDK already supplies `EventTimelineItem.isOwn`). Removed the `client.userId()` lookup at the call site.
8. **Infinite re-fetch on decode failure** (`MatronShared/Sources/ViewModels/ChatViewModel.swift`) — added `failedRequests: Set<URL>` so URLs whose `MediaService.swiftUIImage(for:)` returned nil are remembered and `image(for:)` short-circuits to `nil` on subsequent calls. New test: `test_imageRequest_doesNotLoop_whenMediaServiceReturnsNil`.

**Open follow-ups for the next agent (Tasks 14, 14c, 14d, 14e):**

- The bugbot spec for #8 hinted at a deeper refactor (e.g. making the image cache an actor). The current fix resolves the user-visible loop with the minimum diff; if Phase 3 starts surfacing decode-failed UI affordances, revisit whether `failedRequests` should evict on retry.
- `ComposerViewModel.reportAttachmentError(_:)` is iOS-call-site-specific today. When `MacChatView` (Task 14c) wires drag-and-drop, consider whether Mac drag failures should flow through the same surface or get their own field.
- The fix for #2 picks the first `supportedContentTypes` entry with a preferred extension. If Apple ever returns a parent abstract type before concrete ones, we'd want to skip it — keep an eye on this when iOS 19 lands.
